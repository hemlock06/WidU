# `benchmark_device_configs.py`

> 기기 구성 벤치마크 — 스마트워치만 / 스마트폰만 / 둘다(자이로+가속 IMU 낙상).

## 무엇을 / 왜

같은 낙상을 **(a)워치만 (b)폰만 (c)둘다** 가져왔을 때 탐지 성능이 어떻게 달라지는지 측정.

**왜 UMAFall인가**: UMAFall은 한 trial(같은 낙상 이벤트)을 손목·허리·주머니에서 **동시(동기)**
기록한다. 같은 낙상에 워치(손목)와 폰(허리)이 각각 무엇을 보는지 정확히 짝지어 비교할 수 있는
유일한 셋(다른 단일위치 셋은 불가).

**정직성(핵심)**: 배포 모델(wrist/waist)은 학습에 UMAFall을 포함 → 그냥 테스트하면 인샘플(낙관).
그래서 **GroupKFold(피험자 분리)** — fold마다 테스트 피험자를 제외하고 재학습 후 그 피험자에게만
추론. 모든 trial이 '자기 피험자를 못 본' 모델의 예측을 받는다(주관 누수 없음). 베이스 학습셋은
배포와 동일 위치 매핑: 워치=WEDA+UMAFall-WRIST(타 피험자) / 폰=SisFall+UMAFall-WAIST(타 피험자).

## 어떻게

- UMAFall trial별로 손목·허리 윈도우를 **짝지어** 로드(둘 다 유효한 trial만 → 세 구성을 동일 trial로 공정 비교).
- 피험자 분리 5-fold로 워치·폰 모델 각각 재학습, held-out 피험자에 추론(proba).
- 임계: 워치≥0.30 / 폰≥0.40 (`config.FALL_PROBA_TH_BY_SOURCE`).
- **융합('둘다') = 실제 L5와 동일 OR**: 각 기기 독립 발화, 어느 한쪽이라도 발화 시 알람.
  양쪽 동시 발화 = '교차확인'(고신뢰). → `both = watch | phone`, `corrob = watch & phone`.
- 지표: recall(낙상)·오탐율(ADL)·precision·F1 + 단일기기 놓침을 다른 기기가 건진 비율.

## 결과 요약 (v0.22, UMAFall 617 trial · 낙상 189/ADL 428 · 피험자 18, 피험자분리)

| 구성 | recall | 오탐율 | precision | F1 |
|---|---|---|---|---|
| ① 워치만(손목) | 0.984 | 0.091 | 0.827 | 0.899 |
| ② 폰만(허리) | 0.984 | **0.005** | **0.989** | **0.987** |
| ③ 둘다(OR) | **0.995** | 0.096 | 0.821 | 0.900 |

- 양쪽 동시발화(교차확인): 낙상 184/189 회수 · ADL 동시오탐 **0/428**.
- 보완: 워치 놓침→폰 건짐 2/189 · 폰 놓침→워치 건짐 2/189.
- 해석: 폰(허리)이 본질적으로 정밀(손목=낙상유사 동작 多). 워치 주력 이유는 정확도가 아닌 **가용성**(상시 착용).
  "둘다"의 효익 = OR로 recall 최고(1개만 놓침) + 교차확인으로 0-오탐 고신뢰 채널.
- ⚠️ 한계: 피험자 분리(미지의 *피험자*)지 미지의 *데이터셋*이 아님 → cross-dataset(손목 LODO 0.74)보다 낙관.
  UMAFall=14~68세 시뮬레이션 낙상(실 고령 낙상 아님). 폰 베이스(SisFall 4,505)가 워치(WEDA 1,293)보다 학습량 多.
