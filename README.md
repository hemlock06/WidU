# WidU — 독거 고령자 실시간 위급상황 탐지 AI

WIDYU 앱의 응급알람 기능을 위한 **다계층(multi-layer) 위급상황 탐지 파이프라인**.
스마트워치·폰 센서로 독거 고령자/1인 가구의 위급을 포착해 **가족(보호자)에게 알린다**.
현 배포본(단일 심박 MA 룰, SCRUM-287)을 **대체·확장**한다.

> **설계 근거·의사결정 전체 기록**은 [`docs/WidU_해설문서.docx`](docs/)에 있다(D1~D38 의사결정 로그 + §6 검증 결과). 이 README는 그 요약 + 빠른 시작.

---

## 1. 무엇을 푸는가

독거 고령자에게 위급(낙상·심정지·쓰러짐·배회 등)이 발생했을 때 **빠르고 정확하게 가족에게 알리는 것**. 두 가지가 동시에 중요하다:
- **잘 잡아야**(미탐 = 사람이 위험) — 특히 고령 응급 1위인 **낙상**.
- **헛알람이 적어야**(오탐 = 보호자가 알림을 꺼버림 = 제품 사망). **보호자 신뢰가 제품 생명.**

## 2. 핵심 사실 (WIDYU 팀 자료 전수 검토에서 수렴)

실시간으로 쓸 수 있는 신호는 **심박수(1Hz) + 가속도·자이로(50~200Hz)** 둘뿐.
SpO₂·체온·ECG는 **기록형(실시간 불가)** → 응급이 아니라 **조기경보**로만 사용.
자이로는 워치 + BLE 연동 폰 양쪽에서 수신(위치별 독립 처리).

**전송 계약(2026-06-30 팀 합의)**: 워치=심박+자이로 **1초 배치**, 폰=자이로+위치 **3초 배치**.
센서는 고주파(≥50Hz)로 측정하되 배치 전송 — ★낙상 검출은 **충격 전후 ~2초 고주파 윈도우**가
반드시 배치에 담겨야 동작한다(1샘플/3초 식 희소 전송이면 충격 파형 소실 → 낙상 깨짐 ⇒ 온디바이스 버퍼링·충격창 버스트 필수).

## 3. 5계층 아키텍처

가벼운 신호부터 무거운 판단까지 계층적으로 쌓고, 마지막에 **융합**해 오경보를 억제한다.

| 계층 | 모듈 | 역할 |
|---|---|---|
| 게이팅 | [`widu/gating.py`](widu/gating.py) | 신호 품질·활동맥락(수면/안정/활동)·모션 아티팩트 |
| **L0** | [`widu/l0_safety.py`](widu/l0_safety.py) | 결정적 안전룰 — 임상 하드바운드(서맥<40·빈맥>150·무박동<25)+지속성. 급성을 빠르게(30s) |
| **L1** | [`widu/l1_hr.py`](widu/l1_hr.py) | 개인화 심박 — 개인 baseline(EWMA)·맥락조건부 z·콜드스타트 prior·3분 지속·안정시에만 |
| **L2** | [`widu/l2_fall.py`](widu/l2_fall.py) | 낙상 — 충격→사후 무활동 캐스케이드 + 위치별 RandomForest + 파형-무관 안전망 |
| **L3** | [`widu/l3_behavior.py`](widu/l3_behavior.py) | 행동·위치 — 지오펜스·무활동(12h)·넘어진 뒤 미회복·야간 배회 |
| **L4** | [`widu/l4_trend.py`](widu/l4_trend.py) | 느린 추세 — 안정심박↑·SpO₂↓·체온 CUSUM(감염 조기경보) |
| **L5** | [`widu/l5_fusion.py`](widu/l5_fusion.py) | 융합·등급·에스컬레이션·알림예산·능동 확인(self-check) |
| 통합 | [`widu/pipeline.py`](widu/pipeline.py) | `StreamProcessor` — 사용자별 상태·라우팅 |
| 서빙 | [`serving/api.py`](serving/api.py) | Flask REST(SCRUM-287 후계) |

## 5. 엣지케이스·오경보 처리

실세계에서 오탐/미탐을 부르는 상황과 그 처리(= 왜 신뢰할 만한가).

| 상황 | 위험 | 처리 |
|---|---|---|
| 워치 미착용/방전 | 미탐(침묵) | No-contact/무데이터 → "미착용", 장시간 시 보호자 안내(경보 아님) |
| 충전 중 | 오탐 | 충전 컨텍스트 → 갭 억제 |
| 타앱 운동세션(SCRUM-298) | 측정중단 | 갭 감지 → 로컬푸시 재개 유도 |
| BLE 끊김 / 폰 오프라인 | 미탐 | 워치독 "last seen" + 복구 시 동기화 |
| 모션 아티팩트(세수·설거지) | HR 오탐 | accel 동시성으로 HR 스파이크 감점 |
| 운동(정상 빈맥) | 오탐 | **활동맥락 게이팅**(ACTIVE면 무시) |
| 수면(정상 서맥) | 오탐 | 수면맥락 게이팅 |
| ADL(앉기·눕기·점프) | 낙상 오탐 | **사후 무활동 미충족 → 기각** |
| 워치 떨굼 | 낙상 오탐 | 착용(no-contact) 동시 확인 |
| 신규 사용자(baseline無) | 오탐/미탐 | 모집단 prior+보수 임계 → 점진 적응 |
| 여행·자녀집(새 위치) | 지오펜스 오탐 | 다중 안심구역·임시 학습 |
| 삼성↔애플·단위 차이 | 데이터 불일치 | 단위·주기 정규화, 권역 플래그 |
| 다인 가구(워치 공유) | 신원 혼동 | 1워치=1시니어 가정 명시 |

## 6. 주요 의사결정 (로그 D1~D38 중 핵심)

설계 원칙: **"논문으로 검증 가능 + 과장 없음 + 데이터로 확인."** 핵심 분기:

- **L0/L1 분리** — 급성(실신·서맥·빈맥)은 L0가 하드룰로 30초에, 미묘한 개인 이탈은 L1이 개인화+3분 지속으로. 오탐·미탐을 한 임계로 못 푸니 분리.
- **L1 오탐 해결**(D28~29) — 긴 지속(3분)+안정시에만 → PPG-DaLiA 오탐 **2.06→0/시간**. 지속시간은 스윕으로 최단(knee 115s) 위 마진 둔 3분.
- **낙상은 위치별 모델**(D22) — 허리↔손목 전이 시 민감도 0.91→0.60 급락 → 워치=손목, 폰=허리 분리 필수.
- **일반화는 LODO로 정직하게**(D32) — in-domain CV 0.98은 낙관. 미지 데이터셋 실제 F1 허리 0.90·**손목 0.74**. 주력기기(워치)가 약점 → 단독 신뢰 금지.
- **더 모으면 좋다는 오해**(D32) — 허리 모델에 데이터 추가가 *악화*. 무차별 수집 ≠ 향상. 표적만.
- **단독 무활동 격하**(D30) — 거친 활동신호로 12h 단독 EMERGENCY는 3.5/년 오발 → CAUTION으로, 충격·이상심박 동반 시에만 위급.
- **파형-무관 캐스케이드+네이티브 위임**(D34) — 고령 소프트폴을 분류기가 놓쳐도 *센 충격+무활동*이면 잡음. 애플/삼성 워치 네이티브 낙상(실 낙상 학습)을 받아 융합.
- **능동 확인 루프(self-check)**(D35) — 애매하면 워치가 "괜찮으세요?" → 45초 무응답이면 격상. **사람의 응답을 정답으로** 사용.
- **능동학습 데이터 엔진**(D37) — 위 응답이 곧 라벨. 배포할수록 실 고령 데이터가 쌓여 약점을 자력 개선.

## 7. 원본(WIDYU-ai) 대비 성능 — 머리맞대 검증

현 배포본 [`GB-able/WIDYU-ai`](https://github.com/GB-able/WIDYU-ai)(심박 MA±2σ+고정밴드[80~100]) 알고리즘을 그대로 재구현해 **동일 입력**으로 비교([`scripts/benchmark_vs_baseline.py`](scripts/benchmark_vs_baseline.py)).

**① 실데이터 오경보율 (PPG-DaLiA 건강 15명·35.9h):**

| 시스템 | 오경보/시간 |
|---|---|
| WIDYU-ai(원본) | **54.6** (하루 ~1,300건) |
| **WidU** | **0.5** (≈110배 적음) |

**② 응급 탐지:** 둘 다 5/5 탐지(무승부). 단 원본은 *과발화*로 잡음 — 점진 서맥을 HR이 아직 ~65(정상)일 때 발화. **'HR 65 하락중(정상)'과 'HR 35(응급)'을 구별 못 함** = 54/h 오경보와 같은 현상. 원본의 속도 우위는 **알람 피로로 무의미**(가족이 알림 차단→실 recall≈0). **+원본은 낙상 0 탐지.** → 데이터가 **통째 교체**를 지지.

## 8. 능동학습 — 배포가 곧 데이터 엔진

POC/실 고령 낙상 데이터 없이도 **배포 자체로** 실 라벨을 모은다.
이벤트 → 직전 10초 IMU 캡처 → **사용자 "괜찮아요"(오경보)/가족 사후확인(낙상)** = 라벨 → 디스크 적재 → 재학습.
- 코드: [`widu/datalog.py`](widu/datalog.py), pipeline `respond_ok`/`confirm_incident`
- **프라이버시**: 동의(`collect_data` / env `WIDU_COLLECT=1`) 시에만, 기본 off. 신호+라벨+최소 메타만.

## 9. 설치 · 실행

```bash
pip install -r requirements.txt          # numpy/pandas/scipy/scikit-learn/joblib/flask

# (선택) 공개 검증 데이터셋 받기 — data/ 는 git 제외(대용량)
python scripts/download_data.py --all

python scripts/demo_stream.py            # 5계층 스트리밍 데모
python serving/api.py                    # 서빙 :5001 (능동학습: WIDU_COLLECT=1)

# 낙상 모델 재학습(데이터 받은 뒤)
python scripts/train_fall_final.py --position wrist --smm --elderly
python scripts/train_fall_final.py --position waist

python scripts/reverify.py               # 11라운드 자가검증
python scripts/benchmark_vs_baseline.py  # 원본 대비 머리맞대 벤치마크
```

**경로는 환경 독립** — 모든 경로는 [`widu/config.py`](widu/config.py)의 `ROOT`(repo 루트 상대) 기준. 하드코딩된 머신 경로 없음, 어디서 clone 해도 동작.

## 10. REST API

```
POST /users/<uid>/hr             {ts,bpm,accuracy}
POST /users/<uid>/imu            {ts,ax,ay,az,gx,gy,gz}      # g·rad/s
POST /users/<uid>/location       {ts,lat,lon}
POST /users/<uid>/record         {ts,kind,value}             # kind: SPO2|TEMP|HRV|RESTING_HR
POST /users/<uid>/safezones      {zones:[[lat,lon,radius]]}
GET  /users/<uid>/status
POST /users/<uid>/respond_ok                                 # self-check "괜찮아요"(=오경보 라벨)
POST /users/<uid>/confirm_incident  {is_fall, by}            # 가족 사후확인(=낙상/오경보 라벨)
GET  /collector/stats                                        # 능동학습 수집 통계
```
응답: `{level: NORMAL|INFO|CAUTION|EMERGENCY|NO_CONTACT, reason, escalation, detections}`
(앱 3-상태 안정/주의/위급과 1:1)

## 12. 정직한 한계

- **실 낙상 부재** — 모든 공개셋은 시뮬레이션(자원자). 실 노쇠 고령자 진짜 낙상 아님.
